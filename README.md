# Расчёт НМЦК — Портал Поставщиков

Веб-приложение для расчёта начальной (максимальной) цены контракта (НМЦК) по методу сопоставимых рыночных цен в соответствии с 44-ФЗ.

## Что делает проект

- **Поиск товара/услуги** — вводишь наименование, система находит аналоги в реестре контрактов
- **Расчёт НМЦК** — автоматически считает цену по реальным закупкам, исключает выбросы (IQR), показывает статистику
- **Управление контрактами** — можно вручную включить/исключить отдельные контракты из расчёта
- **Корзина закупок** — добавляй несколько позиций, редактируй количество, пересчитывай
- **Формирование документа** — генерирует обоснование НМЦК в формате `.docx` по каждой позиции отдельно или по всей корзине сразу
- **История** — все сформированные документы хранятся на сервере, можно скачать повторно
- **Обоснование** — структурированный текст по ФЗ-44 приходит с сервера и отображается прямо в приложении

## Стек

- **Vue 3** (Composition API, `<script setup>`)
- **Pinia** — стейт-менеджмент
- **Vue Router 4** — маршрутизация
- **Vite** — сборщик
- **docx-preview** — рендер Word-документов в браузере

## Установка и запуск

```bash
# Клонировать репозиторий
git clone <repo-url>
cd hack130326

# Установить зависимости
npm install

# Создать .env (см. ниже)

# Запустить в dev-режиме
npm run dev
```

Приложение будет доступно на `http://localhost:5173`

## Переменные окружения

Создай файл `.env` в корне проекта:

```env
VITE_API_BASE_URL=https://your-backend-url.example.com
```

| Переменная          | Описание                                 |
|---------------------|------------------------------------------|
| `VITE_API_BASE_URL` | Базовый URL бэкенда (без слеша на конце) |

В dev-режиме Vite проксирует все запросы `/api/*` на этот адрес — CORS не нужен.
В production `VITE_API_BASE_URL` используется напрямую как префикс для fetch-запросов.

## Сборка для production

```bash
npm run build
```

Статика появится в папке `dist/`. Подходит любой статический хостинг или nginx.

> При деплое на nginx настрой `try_files $uri /index.html` — приложение использует History API (не хэш-роутинг).

## Структура проекта

```
src/
```

## API

Бэкенд должен реализовывать следующие группы эндпоинтов:

| Метод | Путь | Описание |
|-------|------|----------|
| `POST` | `/api/v1/auth/login` | Авторизация |
| `POST` | `/api/v1/auth/refresh` | Обновление токена |
| `POST` | `/api/v1/workspace/search` | Поиск по реестру, создание воркспейса |
| `POST` | `/api/v1/workspace/:id/nmck` | Пересчёт НМЦК с фильтрами |
| `GET` | `/api/v1/cart` | Список позиций корзины |
| `POST` | `/api/v1/cart` | Добавить позицию |
| `PATCH` | `/api/v1/cart/:id` | Обновить поля позиции |
| `POST` | `/api/v1/cart/:id/nmck` | Обновить НМЦК позиции |
| `DELETE` | `/api/v1/cart/:id` | Удалить позицию |
| `POST` | `/api/v1/report/single` | Документ по одной позиции (`.docx`) |
| `POST` | `/api/v1/report/cart` | Документ по всей корзине (`.docx`), очищает корзину |
| `GET` | `/api/v1/report/history` | История сформированных документов |
| `GET` | `/api/v1/report/history/:id` | Повторное скачивание документа |
| `DELETE` | `/api/v1/report/history/:id` | Удалить запись из истории |
